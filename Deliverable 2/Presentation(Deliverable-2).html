Key Takeaway: A data-structure-driven foundation enabling behavioral fingerprinting and anomaly detection.

Problem Statement

Modern cybersecurity systems rely on static signatures that miss insider threats. Behavioral anomalies such as unusual sequences or abnormal timing can indicate malicious activity.

  • Timing Anomalies: abnormal process execution speeds.
  • Sequence Anomalies: unusual command or access patterns.
  • Insider Threats: hidden in legitimate behavior.

Solution Architecture

  • Array_Handler: Baseline behavior tracking.
  • Circular_Queue: Real-time sequence monitoring.
  • Linked Lists: Session logging and playback.
  • Pointer_Utils & File_IO: Stability and persistence.

Code Integration

if (dataTransfer > dailyThreshold) {
    logAnomaly();
    alertAdmin();
    haltWriteOps();
}

Modules like StackMonitor, QueueMonitor, ArrayHandler, and PointerUtils work together to detect and log anomalies early.

Conclusion & Future Work

  • Strong, stable data structure core for anomaly detection.
  • Next: Integrate hash tables and trees for advanced fingerprinting.
  • Goal: Precise, real-time insider threat detection.
